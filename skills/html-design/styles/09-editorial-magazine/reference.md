# 杂志编辑风参考资料

## 来源

- Nielsen Norman Group: https://www.nngroup.com/articles/how-people-read-online/
- Apple Human Interface Guidelines Typography: https://developer.apple.com/design/human-interface-guidelines/typography

这些 URL 仅用于人工溯源。以下内容已经把来源思路本地化，agent 运行时不要访问外部 URL。

## 本地化来源笔记

### NN/g How Users Read on the Web

- 长内容读者常扫描，标题、段首、重点句、列表和图注要承载主要信息。杂志风页面不能只追求视觉排版。
- 有信息量的小标题比机巧标题更可靠。章节标题应说明观点、问题或结论。
- 倒金字塔结构适合网页阅读：先给结论和角度，再展开背景、证据、故事和延伸内容。
- 段落应短，一段一个观点。pull quote 可以强化观点，但数量过多会打断阅读路径。

### Apple HIG Typography

- 排版选择应表达层级和品牌气质，同时保证正文可读。杂志风可以使用更有性格的标题字体，但正文需要稳健。
- 字体组合最多两套。标题、导语、正文、图注、引用和栏目名应形成可预测的尺度关系。
- 留白、栏宽和行高决定阅读质量。双栏和图文穿插只适合桌面端，小屏应回到单栏阅读。

## 可提炼设计原则

- 杂志编辑风通过大标题、栏目节奏、图文穿插、章节编号、引文和留白形成阅读张力。
- 这类页面适合内容叙事、报告、个人品牌、案例研究和深度说明。视觉重点是建立阅读路径，而不是把所有内容卡片化。
- 标题要承担信息量。读者只扫标题、导语、图注和重点句时，也应理解页面主线。
- 图片要具体且有编辑价值。人物、产品、现场、历史素材、数据图和过程图比纯装饰图更适合。
- 字体组合可以更有性格，但全站最多两套字体。正文可读性始终优先。

## 版式和组件

- 首屏像封面：强标题、短导语、主题标签、署名或日期，以及一个能说明主题的主视觉。
- 正文使用舒适栏宽，搭配章节编号、细分割线、pull quote、图注和重点句形成节奏。
- 长内容拆成小节，每节有前置信息的标题和短段落。
- 移动端转为单栏，保留章节编号、图注和标题层级，删除会压缩正文的复杂双栏。

## 常见失误

- 只放大标题，没有形成完整阅读结构。
- 引用块和装饰过多，打断主叙事。
- 图像与主题关系弱，页面像通用模板。
- 正文栏宽太宽或太窄，阅读体验下降。
