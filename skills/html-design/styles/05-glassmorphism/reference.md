# 玻璃拟态风参考资料

## 来源

- W3C WCAG 2.2 Contrast Minimum: https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html
- web.dev Web Vitals: https://web.dev/articles/vitals

这些 URL 仅用于人工溯源。以下内容已经把来源思路本地化，agent 运行时不要访问外部 URL。

## 本地化来源笔记

### WCAG Contrast Minimum

- 玻璃拟态最大风险是文字和背景互相穿透。正文需要在最终背景图层上满足对比，而不是只看面板色值。
- 透明面板应有足够遮罩、边框和局部实色层。重要标题、CTA、价格、表单和错误信息不要直接压在复杂背景上。
- 颜色和模糊不能替代明暗对比。深色背景上的浅色字、亮色背景上的深色字都要考虑背景局部变化。

### web.dev Web Vitals

- Web Vitals 把页面体验拆成加载、交互和视觉稳定性。玻璃拟态常用大图、模糊、光效和动画，需要控制性能成本。
- LCP 关注首屏主要内容出现时间。首屏主视觉应压缩、预留尺寸，关键文案和 CTA 不依赖慢加载图片才能被理解。
- INP 关注交互响应。大量 backdrop-filter、阴影和动画可能拖慢交互反馈，应限制层数和动效范围。
- CLS 关注视觉稳定性。图片、视频、卡片和图表都要预留宽高，避免加载后推挤文字和 CTA。

## 可提炼设计原则

- 玻璃拟态依靠透明填充、背景模糊、细边框和层叠光感制造空间。背景复杂度必须受控，关键文字不能压在高噪声区域上。
- 透明面板需要足够遮罩强度。真实判断标准是正文在最终背景上是否清楚，而不是单独看颜色值。
- 模糊、发光和大图会带来性能成本。首屏关键内容和 CTA 不能依赖慢加载视觉资产才能被理解。
- 这种风格适合 AI、数据、创意工具、未来感产品和高视觉冲击的活动页。信息密集页面需要减少透明层数。
- 静态状态必须完整。即使没有动效，页面也要能表达空间层级、当前状态和操作入口。

## 版式和组件

- 背景使用暗色渐变、低噪声光带、抽象 3D、产品截图或数据纹理。文字区域放在复杂度最低的位置。
- Glass panel 由半透明底色、细边框、轻模糊和内层高光共同组成。单纯降低 opacity 会让文字和背景混在一起。
- 图片、视频和图表使用固定尺寸容器，减少加载后布局跳动。
- 移动端减少叠层数量，优先保留一两个主面板，把其余内容转为实色或半实色模块。

## 常见失误

- 背景过亮或过花，正文需要用户费力辨认。
- 所有卡片都透明，页面缺少稳定阅读层。
- 模糊和光效过多，影响低性能设备体验。
- CTA 没有足够实色承托，看起来像普通装饰。
