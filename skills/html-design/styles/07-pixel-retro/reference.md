# 像素复古风参考资料

## 来源

- W3C WCAG 2.2 Contrast Minimum: https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html
- Nielsen Norman Group: https://www.nngroup.com/articles/how-people-read-online/

这些 URL 仅用于人工溯源。以下内容已经把来源思路本地化，agent 运行时不要访问外部 URL。

## 本地化来源笔记

### WCAG Contrast Minimum

- 深色像素界面常用绿色、青色、粉色和黄色。所有正文、按钮、状态标签和表单说明仍要满足明暗对比，不能只靠复古色相。
- 像素字体边缘硬、字形细节少，长正文实际可读性更低。正文应使用更清楚的等宽或系统字体，并提高行高。
- 状态颜色需要配合文字或图标，例如 error、ready、active、locked、complete，不要让用户只靠颜色判断状态。

### NN/g How Users Read on the Web

- 用户扫描页面时依赖标题、列表、关键词和段首。像素复古页可以做终端感，但命令行块、状态栏和任务卡都要有清楚信息层级。
- 有趣语气不能牺牲理解。按钮标签、教程步骤和产品说明要短句化，避免把核心行动包装成难懂梗。
- 倒金字塔结构适合复古风：先告诉用户任务和结果，再提供 lore、彩蛋或社区氛围。

## 可提炼设计原则

- 像素复古风通过等宽字体、硬边框、低分辨率图标、终端面板和游戏化状态建立识别度。
- 可读性比风格还重要。像素字体适合标题、标签和短数字，长正文应使用更清楚的等宽或系统字体。
- 页面结构应像任务界面：状态、任务、输出、行动入口清楚排列。复古装饰不应打断主流程。
- 色彩可以鲜明，但强调色数量要少。深色底上的绿色、青色、粉色或黄色需要足够明暗对比。
- 适合游戏、开发者工具、开源项目、社区活动和 meme 产品。正式商业服务需要减弱游戏感。

## 版式和组件

- 使用硬切色块、方形按钮、像素边框、状态灯、命令行块和徽章系统。
- 图片可以是 sprite、像素地图、终端截图、游戏化成就图标或低分辨率插画。
- 静态状态也要表达交互性。按钮可通过反色、厚边框、方块光标或状态文字区分。
- 移动端减少多列终端面板，保留核心状态栏、主任务和一个清楚 CTA。

## 常见失误

- 整页使用难读像素字体，正文阅读成本过高。
- 装饰性 terminal 太多，用户找不到真实内容。
- 颜色像游戏界面，但信息层级和转化路径不清楚。
- 小屏保留密集网格，导致横向滚动。
