# 新拟态风参考资料

## 来源

- W3C WCAG 2.2 Contrast Minimum: https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html
- Apple Human Interface Guidelines Typography: https://developer.apple.com/design/human-interface-guidelines/typography

这些 URL 仅用于人工溯源。以下内容已经把来源思路本地化，agent 运行时不要访问外部 URL。

## 本地化来源笔记

### WCAG Contrast Minimum

- 新拟态常用相近背景、高光和阴影营造触感，这会天然降低边界和文本对比。正文、按钮、输入框、错误提示不能依赖阴影来表达可读性。
- 普通文本至少 4.5:1，大文本至少 3:1。浅蓝、浅灰、米白背景上的灰字需要主动加深。
- placeholder、focus 文本、禁用态说明和表单辅助文字也属于用户需要阅读的信息，应按实际背景检查。
- 细字重会因抗锯齿看起来更淡。新拟态页面应使用足够字重和更高对比，而不是追求极轻字体。

### Apple HIG Typography

- 排版选择要同时帮助可读性、信息层级、重要内容传达和品牌气质。柔和视觉不能替代层级设计。
- 字体数量要少，字号阶梯要清楚。主标题、副标题、正文、标签、按钮之间需要通过字号、字重和间距区分。
- 亲和、圆润的字体适合新拟态，但正文仍应保持舒适行高和稳定字宽，避免整页变成低对比软块。
- 控件文字要短而明确。按钮标签、开关标签和输入说明应告诉用户操作结果。

## 可提炼设计原则

- 新拟态依赖相近背景色、高光和阴影模拟柔和凸起。它的美感来自轻微触感，不适合承载复杂信息密度。
- 可读性优先于柔和效果。正文、按钮、表单、错误提示和链接需要足够对比，不应因为低对比阴影而难以辨认。
- 交互控件要有明确状态。按钮不能只靠浮雕感表达可点击性，需要填充、边框、文字色或图标状态辅助。
- 新拟态适合轻量工具、健康、习惯、个人主页、设置面板和小型数据卡。大型营销页需要谨慎使用。
- 字体要圆润、清楚、现代。字号、字重和间距比阴影更能建立信息层级。

## 版式和组件

- 使用浅灰、淡蓝、米白作为背景，组件阴影由一侧浅色高光和另一侧深色阴影组成。
- 凸起效果保留在按钮、卡片、指标块、开关和小容器上。长正文区尽量平面化。
- 表单 label 外置，输入框边界清楚，错误提示使用明显颜色和文本说明。
- 移动端减少多层阴影，只保留主容器浮起效果，保证低亮度屏幕仍能分辨边界。

## 常见失误

- 所有元素都和背景颜色太接近，用户看不出层级。
- 按钮和卡片外观相同，行动入口不明确。
- 大面积低对比灰字影响阅读。
- 把新拟态用于复杂数据页，导致信息密度和可用性下降。
