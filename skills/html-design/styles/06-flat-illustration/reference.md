# 扁平插画风参考资料

## 来源

- Nielsen Norman Group: https://www.nngroup.com/articles/how-people-read-online/
- W3C WCAG 2.2 Contrast Minimum: https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html

这些 URL 仅用于人工溯源。以下内容已经把来源思路本地化，agent 运行时不要访问外部 URL。

## 本地化来源笔记

### NN/g How Users Read on the Web

- 用户通常扫描页面，依赖有信息量的标题、关键词、列表、图注和段首来理解内容。扁平插画页不能把主要信息藏进长段落。
- 一段只表达一个观点，先给结论，再给解释。教育和知识讲解页面尤其适合倒金字塔结构。
- 插画应和文本形成互补：图形展示关系、步骤、角色和状态，文字给出结论、边界和行动。
- 扫读结构要覆盖移动端。小屏上插画可以后置或拆分，核心文字和步骤不能被图片挤掉。

### WCAG Contrast Minimum

- 明亮插画配色常导致浅色文字和图上标签对比不足。图标标签、步骤编号、按钮和正文都要按背景检查。
- 关键文字应使用真实 DOM 文本，而不是嵌入图片。这样可以保留缩放、复制、翻译和辅助技术支持。
- 图像可以承担解释任务，但图像 alt 和旁边文字要说明图中信息，避免读者只能依赖视觉细节。

## 可提炼设计原则

- 扁平插画适合解释步骤、角色、概念关系和前后变化。插画应承担说明任务，避免只提供泛化友好氛围。
- 视觉系统要统一。人物比例、描边、圆角、阴影、色块颗粒度和图标风格应保持一致。
- 文本结构需要可扫描。标题、步骤标签、图注和总结句要携带主要信息，长段落拆成短段和列表。
- 明亮配色需要控制数量。插画可以丰富，正文和按钮仍要有清楚对比和稳定背景。
- 教育、知识讲解、活动报名、轻产品和儿童/家庭场景适合使用这类风格。

## 版式和组件

- 每个内容区可以配一个功能明确的小图解：流程图、角色场景、对比图、示例画面或概念拆解。
- 关键文字使用 DOM 文本。插画里只放短标签或辅助符号，避免把主要内容做进图片。
- 组件边界保持简单，使用实色按钮、轻边框卡片和清楚图标。
- 移动端让正文先行，插画可以后置、缩小或按步骤拆分。

## 常见失误

- 使用通用人物插画，但内容和产品没有关系。
- 插画风格混杂，页面像多个素材包拼接。
- 图中嵌入大量文字，移动端无法阅读。
- 颜色过亮，按钮和正文层级不清楚。
