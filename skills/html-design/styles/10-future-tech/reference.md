# 未来科技风参考资料

## 来源

- W3C WCAG 2.2 Contrast Minimum: https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html
- web.dev Web Vitals: https://web.dev/articles/vitals

这些 URL 仅用于人工溯源。以下内容已经把来源思路本地化，agent 运行时不要访问外部 URL。

## 本地化来源笔记

### WCAG Contrast Minimum

- 深色科技风需要严格管理文字对比。正文、图表标签、指标单位、按钮和说明不能被发光背景、网格和渐变吞掉。
- 荧光色适合状态、路径、指标和 CTA，不适合作为长正文颜色。大段文字应使用接近白色或高对比浅灰。
- 图表和非文本信息也会受到对比问题影响。线条、节点、图例和标签需要在深色背景上清楚可辨。

### web.dev Web Vitals

- 未来科技风常用重视觉效果，必须控制加载、交互和视觉稳定性。首屏大图、粒子、视频、canvas 和复杂图表需要预留尺寸。
- LCP 要求首屏主要内容尽快出现；技术氛围图不应阻塞标题、关键指标和 CTA。
- INP 要求交互响应快；大量实时动效、滤镜和 canvas 更新需要减少或降级。
- CLS 要求布局稳定；数据面板、图表和图片加载前应有固定容器，避免阅读过程中跳动。

## 可提炼设计原则

- 未来科技风使用深色背景、线框、数据面板、流程轨道、网格和少量荧光色建立技术感。
- 科技感应服务信息。指标、流程、架构和状态需要明确标签、单位、趋势和结论，不能只展示抽象光效。
- 深色页面需要格外关注对比。正文、图表标签、说明文字和 CTA 不能被发光背景吞掉。
- 动效和复杂视觉应可降级。静态截图状态必须完整表达信息，且不依赖加载慢的大图。
- 适合 AI、数据产品、开发者工具、安全、自动化、硬件和技术报告类页面。

## 版式和组件

- 主色通常使用青、绿、蓝、紫中的 1 到 2 个，配合灰阶和低透明网格。
- 指标模块要把数字、单位、时间范围、变化方向和解释放在近处。
- 复杂图表配文字摘要，移动端可转为关键数字卡、简化图或辅助表格。
- 背景扫描线、发光块和粒子只放在背景层或视觉锚点上，重要文本放在实色或强遮罩区域。

## 常见失误

- 科技光效很强，但用户看不出产品价值。
- 深色面板层数太多，正文对比不足。
- 图表只展示形状，没有结论和数据口径。
- 大图和重特效拖慢首屏，导致关键内容延迟出现。
