## 本地化来源笔记

### Material Responsive Layout Grid

- 强视觉风格更需要网格。粗边框、硬阴影和旋转元素如果没有 columns、gutters、margins 约束，会快速变成随机堆砌。
- 粗野派页面可以破坏精致感，但不应破坏阅读路径。标题、正文、图片和 CTA 仍要按同一组边界对齐。
- 移动端需要重新计算阴影偏移和横向宽度，避免硬阴影造成水平溢出。

### WCAG Contrast Minimum

- 高饱和色块上的文字不能靠“颜色够亮”判断，需要检查明暗对比。黄色底通常适合黑字，深色底通常适合白字。
- 彩色文字放在彩色底上风险很高。按钮、表单、错误提示和价格标签应优先使用黑/白/近黑文字。
- 颜色不能作为唯一状态提示，必要时加入边框、图标、形状、标签或位置变化。

### NN/g How Users Read on the Web

- 用户通常扫描页面，依赖标题、关键词、列表和段首信息理解内容。新粗野派的大标题应承担真实信息，而不是只承担情绪。
- 每段只表达一个观点，长说明拆成短句和 bullet。强视觉环境下，正文越需要简短、清楚、可定位。
- 倒金字塔结构对强风格页面尤其有用：先给结论和行动，再补充态度、故事和细节。

## 可提炼设计原则

- 新粗野派靠粗边框、硬阴影、大字号和高饱和色块形成记忆点。强烈视觉需要更严格的网格约束，否则会变成随机拼贴。
- 文案要短、直接、有态度。标题可以很大，但副标题仍要说明具体价值和使用场景。
- 色块承担信息分组，不应只是装饰。主色块承载主消息，辅助色块承载步骤、标签、证据或 CTA。
- 高对比不等于高可读。彩色底上的文字需要实际检查明暗差，黑白文字通常比彩色文字更稳定。
- 这种风格适合年轻化产品、创作者工具、社区项目、活动页和开发者玩具，正式企业服务需要降低冲突感。

## 版式和组件

- 使用黑色或近黑粗线作为统一边界，按钮、标签、卡片和图片都遵循同一描边厚度。
- 硬阴影使用固定偏移，例如 4px 或 6px。阴影方向全站一致，避免每个模块有不同投影逻辑。
- 图像可以做贴纸式裁切、粗描边截图、半调纹理或高反差照片，但图像内容仍要说明产品、人物或结果。
- 移动端保留粗边框和强标题，减少旋转、重叠和大阴影，防止产生横向溢出。

## 常见失误

- 每个元素都很夸张，页面没有信息主次。
- 使用多种强色，正文和按钮对比不足。
- 把内容全部做成贴纸，缺少可扫读的结构。
- 小屏上硬阴影和旋转元素撑破视口。
