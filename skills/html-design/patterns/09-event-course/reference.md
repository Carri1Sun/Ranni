# 活动 / 课程招募页参考资料

## 来源

- Nielsen Norman Group Website Forms Usability: https://www.nngroup.com/topic/forms/
- Nielsen Norman Group Homepage Design Principles: https://www.nngroup.com/articles/homepage-design-principles/

这些 URL 仅用于人工溯源。以下内容已经把来源思路本地化，agent 运行时不要访问外部 URL。

## 本地化来源笔记

### NN/g Form Error and Form Usability Guidance

- 报名流程需要低阻力。字段只保留完成报名需要的信息，其他调研问题可以后置。
- label、错误提示、成功状态和隐私说明要清楚，尤其是手机号、邮箱、发票、优惠码、付款状态等字段。
- 如果报名失败，错误信息要说明问题和恢复方式，避免用户重复提交或离开。
- 提交成功后要说明票据、课程链接、提醒邮件、资料获取、退款规则和下一步。

### NN/g Homepage Design Principles

- 活动/课程首屏需要表达主题、适合人群、时间地点、核心收获和报名入口。
- 示例内容可以是课程大纲、作品产出、讲师案例、往期照片、作业样例或学员反馈。
- 页面应主动处理顾虑：适合基础、回放、资料、设备、退款、席位、发票和社群支持。

## 可提炼设计原则

- 活动和课程页面要让用户快速判断是否适合自己，以及报名后能得到什么。
- 首屏应包含主题、适合人群、时间地点、核心收获和报名 CTA。关键信息不要分散到页面深处。
- 大纲应按学习路径或活动流程组织，强调练习、产出物、资料、回放、社群和证书等实际收获。
- 讲师介绍需要说明可信度来源。经历、作品、客户、研究、教学成果或现场照片都可作为证据。
- 价格、席位、退款、发票、回放和设备要求是报名前常见顾虑，应主动处理。

## 结构建议

- Hero：主题、适合人群、时间地点、核心收获、报名按钮。
- 收获：用户能带走的结果，例如模板、作品、清单、案例拆解。
- 大纲：模块、练习、产出物和完成后能力。
- 讲师：可信度、经验、作品和头像/现场图。
- 价格和报名：早鸟、席位、退款、发票和报名表单。
- FAQ：基础要求、回放、资料、设备、退款、适合人群。

## 常见失误

- 只描述讲师想讲什么，缺少学员产出。
- 时间地点、价格和报名入口隐藏太深。
- 讲师介绍没有证据，只写抽象头衔。
- FAQ 缺失，用户报名前反复查找关键信息。
