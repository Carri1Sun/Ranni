## 本地化来源笔记

### NN/g UX Design Portfolios

- 作品集的价值在于展示完整能力，不只是最终视觉。案例需要说明问题、约束、角色、过程、决策、结果和复盘。
- 招聘者或客户会寻找证据：你如何理解问题、如何取舍、如何验证结果、如何与团队协作。
- 视频、原型链接和大图可以作为补充，但不能替代清楚的案例叙述。
- 案例数量应少而强，优先展示最能匹配目标受众的项目。

### Material Responsive Layout Grid

- 项目网格需要 columns、gutters、margins 支撑，确保封面、标签、项目名、年份和角色信息能稳定比较。
- 桌面端可以用多列卡片展示筛选，移动端应单列呈现代表项目和案例摘要。
- 图片比例、卡片高度和文本起点要统一，帮助读者快速比较项目，而不是被图墙淹没。

## 可提炼设计原则

- 作品集要展示能力和判断力。精选少量强案例比堆很多项目更有效。
- 每个案例需要说明问题、约束、个人角色、过程、关键决策、结果数据和复盘。
- 项目网格负责筛选，案例详情负责证明能力。卡片应包含名称、领域、角色、年份和一句结果。
- 视觉资产要有信息量。过程草图、界面演进、上线截图、实物照片、架构图和成果指标比漂亮封面更有证明力。
- 顺序应服务目标受众。求职、接单、品牌展示、摄影集和开发作品集的排序逻辑不同。

## 结构建议

- 封面介绍：作品集定位、目标领域、最强能力。
- 精选项目网格：3 到 5 个代表案例。
- 项目详情：背景、角色、过程、决策、结果和链接。
- 结果数据：指标、影响范围、客户反馈或上线状态。
- 联系入口：读者判断能力后能立即行动。

## 常见失误

- 图片很多但缺少上下文，读者不知道你贡献了什么。
- 所有项目等权，代表性不清楚。
- 案例过程缺失，只展示最终视觉。
- 移动端图墙过长，说明文字被淹没。
