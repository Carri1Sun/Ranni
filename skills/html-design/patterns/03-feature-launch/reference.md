# 功能发布页参考资料

## 来源

- Nielsen Norman Group Homepage Design Principles: https://www.nngroup.com/articles/homepage-design-principles/
- Nielsen Norman Group How People Read Online: https://www.nngroup.com/articles/how-people-read-online/

这些 URL 仅用于人工溯源。以下内容已经把来源思路本地化，agent 运行时不要访问外部 URL。

## 本地化来源笔记

### NN/g Homepage Design Principles

- 功能发布页要立即说明新功能是什么、改变了什么、谁会受益、用户下一步能做什么。
- 示例内容非常关键。截图、动图帧、流程图、before/after 和实际输出能帮助用户判断功能是否值得尝试。
- 页面应提供行动和导航入口，例如开启功能、查看文档、升级计划、提交反馈或查看兼容说明。

### NN/g How Users Read on the Web

- 用户倾向扫描页面，小标题要携带信息。避免只写“能力一”“流程”“适用人群”，应写出变化和收益。
- 前置信息能提高理解效率。页面先讲结果，再解释机制、步骤和边界。
- 长说明应拆成列表、对比块或步骤。每段只讲一个变化点，避免把发布说明写成内部 changelog。
- 重点词、截图标注、步骤编号和短结论可帮助用户在扫读中抓住价值。

## 可提炼设计原则

- 功能发布页应该聚焦一个新能力。页面目标是让已有或潜在用户理解变化、收益和使用入口。
- 解释新功能时，前后对比通常比单纯功能描述更有效。用户需要看到旧流程痛点和新流程结果。
- 发布页需要展示操作路径。入口在哪里、如何配置、执行后会得到什么结果，都应通过截图、步骤或 demo 帧说明。
- 标题和小标题要前置信息。很多用户只扫标题和图片，就应该理解新功能的价值。
- CTA 文案要对应真实动作，例如打开新版、试用自动摘要、创建第一个规则或查看模板。

## 结构建议

- 功能名称和一句价值主张。
- 前后对比：旧方式、新方式、节省步骤、减少错误或提升结果。
- 核心能力：最多 3 个，配截图或状态变化。
- 操作流程：3 到 5 步，从入口到结果。
- 适用人群：按角色或场景说明收益。
- 体验入口和升级说明。

## 常见失误

- 同一页发布多个功能，用户抓不到重点。
- 只写“更快、更智能”，缺少具体变化。
- 截图没有标注关键位置，用户不知道看哪里。
- CTA 太泛，不能引导用户实际使用新能力。
