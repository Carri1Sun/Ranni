## 本地化来源笔记

### NN/g How Users Read on the Web

- 知识页读者通常会扫描，寻找定义、结论、步骤、例子和总结。内容应通过信息量明确的小标题组织。
- 倒金字塔适合解释类页面：先给核心定义和结论，再解释背景、机制、案例、误区和延伸。
- 段落要短，一段一个观点。关键术语首次出现时解释，后续使用统一称呼。
- 列表、重点句、图注和示例比长段落更容易被读者吸收。

### web.dev Responsive Web Design Basics

- 响应式页面要设置适配视口，避免固定宽度元素导致横向滚动。
- 图片设置 `max-width: 100%` 可以防止超过容器；同时应提供宽高，让浏览器在图片加载前保留空间，减少布局跳动。
- 图表、公式和代码块需要小屏策略。可以使用简化图、摘要卡、横向滚动代码块或分段展示，但页面整体不能横向溢出。
- 断点应由内容决定。复杂解释页不应为了桌面双栏而牺牲手机阅读顺序。

## 可提炼设计原则

- 知识讲解页服务理解。开头要先回答这个主题为什么重要，再给出一句话定义。
- 结构宜采用倒金字塔：结论和核心概念先出现，背景、细节、边界和延伸阅读放在后面。
- 用户常常扫描内容。标题、重点句、图注、列表和总结卡需要承载主信息。
- 复杂概念需要多种表达：类比、步骤、图示、示例、对比表和常见误解。
- 响应式阅读体验要稳定。图片和图表自适应，正文不横向溢出，断点根据内容复杂度决定。

## 结构建议

- 主题介绍：问题、价值、一句话定义。
- 背景问题：用户为什么会遇到这个概念。
- 核心概念：拆成 2 到 4 个清楚小节。
- 分步骤解释：按因果、流程或层级展开。
- 示例：真实案例、代码、公式或场景图。
- 总结和延伸阅读。

## 常见失误

- 开头铺垫过长，用户很久看不到定义。
- 小标题只有“概念一”“步骤二”，缺少信息。
- 图示没有文字解释，用户必须自己推断。
- 移动端图表过宽，导致整页横向滚动。
