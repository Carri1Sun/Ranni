- 首屏说明教程目标、适合人群、预计耗时、准备条件和完成后的检查标准。
- 步骤导航要清楚，建议使用编号、进度条或侧栏目录；每一步只承担一个操作任务。
- 每一步包含操作说明、代码块或截图、预期结果和常见错误；代码块要可复制并避免移动端水平溢出。
- 截图要标出点击区域、菜单路径或配置值；图片旁保留文字说明，避免只靠截图传达关键步骤。
- 表单、配置项和命令参数要有 label、默认值解释和错误恢复提示，不把关键说明藏在 tooltip 里。
- 复杂步骤可以加入检查点，例如“如果看到 200 OK，继续下一步”，帮助用户自查。
- 完成检查放在教程末尾，列出产物、命令输出、页面状态或截图结果。
- 移动端步骤单列排列，代码块可横向滚动但页面整体不能横向滚动。
- 对危险操作、覆盖文件、付费资源或外部账号授权要提前提示，并提供保守替代路径。
- 步骤标题应描述结果，例如“创建 API key”或“验证部署成功”，帮助用户从导航中快速定位进度。
