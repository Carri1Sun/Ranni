# 交互式教程页参考资料

## 来源

- Nielsen Norman Group How People Read Online: https://www.nngroup.com/articles/how-people-read-online/
- Nielsen Norman Group Website Forms Usability: https://www.nngroup.com/topic/forms/

这些 URL 仅用于人工溯源。以下内容已经把来源思路本地化，agent 运行时不要访问外部 URL。

## 本地化来源笔记

### NN/g How Users Read on the Web

- 教程用户在执行任务时会反复扫描和定位。步骤标题、编号、检查点和预期结果要比普通文章更明确。
- 每一步只讲一个操作目标，先说要得到什么结果，再给命令、截图、配置值和解释。
- 倒金字塔也适用于教程：先给目标、前置条件和完成标准，再进入细节。
- 代码、截图、警告和常见错误应靠近对应步骤，减少用户上下跳转。

### NN/g Form Error and Form Usability Guidance

- 配置表单、命令参数和输入字段都需要清楚 label、默认值解释和错误恢复方式。
- 错误提示应靠近问题来源，告诉用户如何修正。不要只写“失败”或把错误处理集中到文末。
- 复杂字段可以使用实时反馈，例如密码规则、API key 格式、URL 可访问性或权限状态。
- 对危险操作、覆盖文件、付费资源或授权行为要提前提示，并提供确认或保守路径。

## 可提炼设计原则

- 教程页的目标是让用户完成任务。页面需要清楚说明准备条件、预计耗时、步骤顺序和完成标准。
- 每一步只承担一个操作任务。步骤应包含操作说明、代码或截图、预期结果和错误恢复。
- 用户在执行教程时经常来回跳转。编号、进度、侧栏目录、锚点和完成检查可以降低迷路概率。
- 表单、配置项和命令参数要有 label、默认值解释和错误提示。不要把关键说明只放在 placeholder 或 tooltip。
- 代码块和截图需要移动端策略。代码块可横向滚动，但页面整体不能横向滚动。

## 结构建议

- 教程目标、适合人群、预计耗时和准备条件。
- 步骤导航或目录。
- 每一步：目标、操作、代码/截图、预期结果、检查点。
- 常见错误：错误表现、原因和修复方式。
- 完成检查：命令输出、页面状态、文件路径或截图结果。

## 常见失误

- 步骤太长，一步里包含多个操作。
- 截图没有标注点击位置或配置值。
- 缺少预期结果，用户不知道自己是否做对。
- 错误处理被放到文末，用户遇到问题时难以恢复。
