- 页面要短，首屏用一句话说明产品承诺、目标人群和当前阶段，让用户马上知道为什么要留下邮箱。
- 邮箱表单字段保持极少，默认只收邮箱；label 外置，placeholder 只写示例，按钮说明用户会获得内测、通知或名额。
- 在表单前提供足够价值信息，例如 3 个亮点、上线节奏、demo 截图、原型动图或正在解决的问题。
- 亮点模块只保留 3 个以内，每个亮点用用户收益表达，避免提前写完整产品文档。
- 进度或路线图用简短状态表达，例如“封闭测试”“公开候补”“预计上线”，帮助用户判断可信度。
- 社交证明可以是等待人数、早期用户画像、合作方、创作者头像或简短引用，但不要制造虚假热度。
- 表单附近提供隐私说明和提交后反馈；错误提示要指出问题并允许用户直接修正。
- 移动端把表单放在首屏可见范围内，亮点和证明下移，避免用户滚动很久才看到行动入口。
- 页面可以展示一个小型 demo、原型截图或使用场景，让用户理解产品方向，不要只靠神秘文案制造期待。
- 如果使用倒计时、名额或进度信息，必须让它服务可信度和行动紧迫感，避免看起来像虚假营销组件。
