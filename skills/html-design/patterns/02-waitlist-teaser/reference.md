# 等候名单 / 预告页参考资料

## 来源

- Nielsen Norman Group Website Forms Usability: https://www.nngroup.com/topic/forms/
- Nielsen Norman Group Homepage Design Principles: https://www.nngroup.com/articles/homepage-design-principles/

这些 URL 仅用于人工溯源。以下内容已经把来源思路本地化，agent 运行时不要访问外部 URL。

## 本地化来源笔记

### NN/g Form Error and Form Usability Guidance

- 表单要降低填写成本。等候名单页通常只需要邮箱；如果收集姓名、角色、公司或需求，需要解释用途。
- label 应保持可见，placeholder 只作为示例。用户开始输入后仍应知道字段含义。
- 错误提示应靠近对应字段，语言具体、礼貌、可执行。不要只在顶部放错误汇总，也不要只用 tooltip 提示错误。
- 可以使用内联验证，但不要在用户还没输完时过早报错。复杂字段可提供成功状态，简单字段不需要过度反馈。
- 提交后要给明确结果：是否加入成功、下一步会收到什么、多久会联系、如何取消订阅。

### NN/g Homepage Design Principles

- 预告页也需要说明身份、用途、样例和行动。神秘感不能替代价值说明。
- 页面要展示足够具体的内容样例，例如原型截图、demo 帧、目标任务、进度状态或用户收益。
- 行动入口要靠前。用户无需阅读完整页面，也应能理解为什么留下邮箱。

## 可提炼设计原则

- 等候名单页要短、明确、可信。用户通常只愿意投入很少时间判断是否留下联系方式。
- 页面需要在表单前给出足够价值：面向谁、解决什么问题、为什么现在值得关注、上线或内测处于什么阶段。
- 表单负担越低越好。默认只收邮箱，必要字段需要说明理由，提交后给出明确反馈。
- 社交证明可以提升可信度，但必须具体可信，例如早期用户画像、等待人数、合作方、创作者背景或真实引用。
- 预告页可以有期待感，但不能只靠神秘感。至少要展示产品方向、亮点和下一步承诺。

## 结构建议

- 首屏：一句强承诺、目标人群、当前阶段、邮箱表单和提交反馈。
- 亮点：最多 3 个，每个亮点讲用户收益。
- 进度：用简短时间线或状态说明内测、候补、公开发布。
- 证明：等待人数、早期用户、创作者头像、合作方或 demo 截图。
- 隐私：表单附近说明邮件用途和退订方式。

## 常见失误

- 页面只写 coming soon，缺少价值信息。
- 表单字段太多，用户在不了解产品前就被要求填写详细资料。
- 提交后没有反馈，用户不知道是否成功加入。
- 虚假热度或过度夸张文案削弱信任。
