# HTML to PPTX Agent 设计与排版执行规范

目标受众是负责生成幻灯片 HTML 代码的 coding agent。agent 输出 HTML/CSS，随后通过 `dom-to-pptx` 或类似 DOM 映射工具转换为原生 PPTX。核心目标是让 HTML 具备稳定的平面设计质量，同时保证 DOM 映射到 PPTX 时有较高保真度和兼容性。

## 1. 容器与画布约束

为了保证 HTML 在 16:9 画幅下适配 PPTX 尺寸，必须严格控制外层容器。

### 必须执行

- 固定视口：必须设定一个全局外层容器，例如 `.slide-container`，并锁定尺寸为 `width: 1280px; min-height: 720px;`。
- 盒模型：全局应用 `box-sizing: border-box;`。
- 溢出控制：在外层容器使用 `overflow: hidden;`，确保装饰元素不会撑破画布。
- 高度控制：内部内容块使用 `min-height`，避免文本过多时被固定高度截断。

### 禁止事项

- 禁止在外层容器或主要内容流中使用 `padding-bottom`。使用 `padding-top` 和 `margin-bottom` 控制垂直间距，避免映射 PPTX 时整体高度计算出错。
- 避免对外层容器使用 `align-items: center` 做全局垂直居中。多页幻灯片应通过固定 `padding-top` 对齐主标题。

## 2. 视觉美学与排版

幻灯片是静态演示媒介，应追求平面设计和杂志排版感。

### 必须执行

- 字体栈保持极简：每套幻灯片最多使用 2 种字体，优先使用统一的无衬线几何字体，例如 `Space Grotesk`、`Inter`、`Montserrat`。
- 字号层级保持稳定：封面大标题约 `64px`，内容页标题约 `32px`，重点副标题或强调数据 `24px` 到 `28px`，正文 `16px` 到 `20px`。
- 留白充足：文本行高建议 `1.5` 到 `1.6`。内容块之间的 `gap` 或 `margin-bottom` 至少 `30px`。
- 水平布局优先：多条并列信息使用 CSS Grid 或 Flexbox 横向展示。

### 禁止事项

- 避免网页 UI 感组件。卡片圆角不得超过 `8px`，不要使用明显 `box-shadow` 制造卡片阴影。
- 禁止 CSS 动画、`@keyframes`、`transition` 和 `:hover`。幻灯片是静态产物，动画会干扰截图或解析引擎。
- 每页正文段落不应超过 3 段，每段不超过 3 行。长文本应提炼为图标列表、小标题块或摘要句。

## 3. 装饰元素与色彩

### 必须执行

- 每页除黑、白、灰背景外，最多使用 2 到 3 种主题色。
- 背景几何图形使用 `position: absolute; z-index: 1;`，避免干扰正常文本流解析。
- 可以在固定位置放置低透明度页码、色带、几何块或锚点，提升平面设计秩序。

### 禁止事项

- 不要在 HTML 中手写复杂内联 SVG 路径。需要复杂插图时使用标准 `<img>` 标签或局部截图回退。
- 避免复杂 CSS 渐变。优先使用纯色、分区底色或极简单向线性渐变。

## 4. DOM 映射与 PPTX 兼容

### 必须执行

- 层级结构明确：背景层使用 `z-index: 1`，主内容层使用 `z-index: 10`。
- 所有 `<img>` 或图片容器必须具有明确像素宽高，并设置合适的 `object-fit`。
- 使用标准 Web 字体或系统字体，并提供安全 fallback。

### 禁止事项

- 包含标题、段落、列表、表格的主内容区不要使用 `position: absolute`。主内容应使用 Flexbox、Grid 或标准文档流。
- DOM 结构保持扁平，避免超过 4 层的深层 Flex/Grid 嵌套。
- 不要用 `::before` 或 `::after` 的 `content` 注入关键文字。伪元素只可用于纯装饰。

## 5. 组件建议

- 图标库可使用基于类名或字体的图标。确保它们在导出失败时仍有可读替代。
- Chart.js、D3 或 canvas 视觉必须放在固定像素宽高的父容器中，并使用局部截图回退。
- 表格使用标准 `<table>`、`<tr>`、`<td>` 结构。避免用 Flexbox 模拟复杂表格。
